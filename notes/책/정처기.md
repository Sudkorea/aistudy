## 요구분석
단계

| 절차             | 설명                                                                                                             |
| -------------- | -------------------------------------------------------------------------------------------------------------- |
| 요구사항 분류        | 요구사항의 유형 확인하는 단계<br>소프트웨어에 미치는 범위 파악<br>생명주기의 변경 파악                                                            |
| 개념 모델링 생성 및 분석 | 모델 : 요구사항을 더 쉽게 이해할 수 있도록 현실세계의 상황 단순화<br>모델링 표기를 위해 쓰는 방법론 : **DFD(Data Flow Diagram), UML 다이어그램, E-R 다이어그램** |
| 요구사항 할당        | 요구사항을 위한 아키텍처 구성요소 식별하는 단계                                                                                     |
| 요구사항 협상        | 각 이해관계자의 요구가 상충되는 상황 방지                                                                                        |
| 정형 분석          |                                                                                                                |
### 요구사항 분석 기술

 

| 분석기술   | 설명                                        |
| ------- | ----------------------------------------- |
| 청취      | 이해관계자의 의견 듣기                              |
| 인터뷰와 질문 | 이해관계자의 정보 수집, 이야기 나누기(이거랑 위에꺼 차이가 있긴한가)   |
| 분석      | 추출된 요구사항 충돌, 중복, 누락 등 분석을 통해 완전성과 일관성 확보  |
| 중재      | 이해관계자들의 상반된 요구에 대한 중재기술                   |
| 관찰      | 사용자 작업 관찰하며 사용자가 언급하지 않은 미묘한 의미 탐지        |
| 작성      | 문서 작성                                     |
| 조직      | 수집된 방대한 정보 -> 일관성있게 바꾸기                   |
| 모델 작성   | 제어의 흐름, 기능처리, 동작행위, 정보 내용 이해하기 쉽게 모델로 바꾸기 |
### 사용하는 모델링 기법

#### DTD(데이터 흐름도)
각 프로세스를 따라 흐르며 변환되는 모습을 나타냄.
시스템 분석과 설계에 매우 유용함
모델링 도구로서 가장 보편적으로 사용됨.
자료 흐름 그래프 또는 버블차트라고 함.

특징
- 구조적 분석 기법에 이용
- 데이터의 흐름에 중심을 두는 분석용 도구
- 제어의 흐름은 중요하지 않음
- 시간 흐름을 명확하게 표현할 순 없음.

구성요소

| 구성요소    | 설명                                                        | 표기  |
| ------- | --------------------------------------------------------- | --- |
| 처리기     | 입력된 데이터를 원하는 형태로 변환하여 출력                                  | 원   |
| 데이터 흐름  | DFD의 구성요소(프로세스, 데이터 저장소, 외부 엔터티)들 간의 주고받는 데이터 흐름을 나타내는 요소 | 화살표 |
| 데이터 저장소 | 데이터가 저장된 장소<br>평행선 안에 데이터 저장소 이름 넣기                       | 평행선 |
| 단말      | 프로세스 처리 과정에서 데이터가 발생하는 시작, 종료 나타냄                         | 사각형 |

### 자료사전
- 개념
	자료 요소, 집합, 흐름, 저장소의 의미, 관계, 그 값, 범위, 단위를 구체적으로 명시함
	파일 혹은 데이터베이스에 있는 자료에 대한 자료 또는 각 자료 항목에 주어진 이름과 길이, 서술과 같은 데이터를 포함하는 참조를 위한 작업
- 목적
	조직에 속한 다른 사람들에게 특정한 자료 용어가 뭘 의미하는지 알려주기 위함.
	DFD의 모든 데이터는 사전에 등록되어 있어야함.
