# Introduction and motivation

TDA는 최근 부상했음. 응용 대수위상과 computational topology 분야이고, 이게 위상이랑 기하가 강력한 질적(가끔 양적)인 추론 접근을 줌.(2017년 논문에 나옴)
데이터를 유클리드 평면과 같은 거리공간에 점으로 이루어진 구름으로 놓는거임. 최근들어 구현되어 연구되는 분야라 하는데, 이쪽은 대부분 간단한 소개임
## TDA Pipeline
많은 사람들이 많은 방법으로 구현해놨으나, 하나의 베이스라인을 따라가는 경향이 있음. 

1. 입력은 유한개의 점군이고, 이 점들은 어떤 거리공간(metric space) 안에 존재한다. 여기서 metric의 선택은 데이터의 기하적·위상적 특징을 결정하는 데 중요한 역할을 한다.

2. 점군 위에 단순복합체(simplicial complex)가 구축되며, 이를 스케일에 따라 확장하면 nested 구조(= filtration)가 된다. 단순복합체는 이웃 그래프의 고차원 일반화이며, 이렇게 구축된 구조는 데이터의 다양한 스케일에서 위상적 정보를 반영함. 이는 많은 표준 데이터 분석이나 학습 알고리즘에서 데이터를 기반으로 고전적으로 구축됨. 이 데이터에서 어떤 유의미한 연관 정보를 어떻게 잘 뽑아내느냐가 과제임

3. 이렇게 얻어진 구조로부터 위상적/기하적 정보를 추출한다. 대표적으로 삼각화(triangulation), 근사(approximation), 요약(summary), 퍼시스턴스 호몰로지(persistent homology) 같은 기법이 사용된다. 또한, TDA는 데이터의 작은 잡음에 대해서도 결과가 크게 변하지 않도록 하는 안정성(stability) 과 이를 위한 통계적 방법론을 중요하게 다룬다.

4. 이렇게 뽑아낸 정보들은 새로운 인사이트를 제공하고, 데이터를 더 잘 이해하게 해줌-특히 시각화라던가,데이터 해석 부분에서. 이 특징은 머신러닝 피쳐로 사용 가능하기도 함.
