

Side information의 정의

Cold start problem : data가 충분하지 않아 사용자와 아이템의 latent factor를 학습할 수 없는 경우

Temporal evolution : 앞서 정의한 compatibility function $f(u,i)$는 시간의 흐름에 따른 사용자 선호도 및 아이템 특성의 변화를 반영하지 않음.(시간축 변수가 필요하단 소리)

왜 추천했는지 설명해야 하거나, 친구 추천, 제약조건 있는 추천 등 cf가 들어맞지 않는 경우가 많음

# Content based Methods
사용자 정보(성별, 연령, 직업) 받아서 cold start를 좀 완화시킴
장점 : cold start 완화에 효과적임
단점 : 비교적 뻔한 추천, 새로운 사용자에 대한 추천은 여전히 한계가 있음

CF와 CB는 각각의 장단점이 상호보완적인 측면이 있어서, 두 방법을 합쳐서 쓰이기도 함.

# Context aware Recommendations
추천이 이루어지는 특정한 상황을 나타내는 추가 정보에 따라 추천 결과를 생성하고 조정함.(이를 context라 부름.)
ex) Time, Location, Social Information등
