## 📌 발표 PPT 슬라이드 구성 (20분)

총 슬라이드 수는 **12~15장 내외**로 구성 추천.

### **표지 (1장)**

- 발표 제목  
    _"금융상품 추천 시스템 기존 모델 분석 결과"_
    
- 발표자 이름 및 소속
    
- 발표 날짜
    

---

### **1. 분석 개요 (1장)**

- 분석 목적
    
    - 레거시 코드 현황을 파악하여 서비스의 동작 원리를 이해
        
- 분석 기간 및 범위
    
    - 분석 기간: 최근 1개월
        
    - 분석 범위: **모델 파트 (추천 알고리즘 중심)**
        

---

### **2. 기존 추천 시스템 구성 (1장)**

- 기존 추천 시스템은 크게 **데이터 / 아키텍쳐 / 모델** 3가지 파트로 구성됨
    
- 각 파트별 간략한 설명 (아주 간략하게 1문장씩)
    
    - 데이터: 상품, 고객 데이터 수집·가공
        
    - 아키텍쳐: 추천 서비스의 데이터 흐름 및 구성요소 관리
        
    - 모델: 추천 알고리즘을 활용해 고객에게 최적 상품 제시
        

> _“본 발표는 ‘모델’ 파트에 집중합니다.”_

---

### **3. 분석 대상 과제 소개 (1장)**

- 총 5개 과제, 이 중 실제 사용된 추천 모델은 3개임
    
    |과제|추천 목적|사용 모델|사용 여부|
    |---|---|---|---|
    |1. 상품군 보유고객 분석 기반 추천|상향추천 (Up-selling)|K-means, ALS, BPR|✅|
    |2. 접촉정보 분석 기반 고객 관심상품 추천|-|미사용|❌|
    |3. 구매패턴 분석 기반 교차추천|교차추천 (Cross-selling)|ALS|✅|
    |4. 신규고객 분석 기반 상품 추천|-|미사용|❌|
    |5. 카드사용패턴 분석 기반 체크카드 추천|사용패턴 기반 추천|ALS|✅|
    

---

### **4. 사용된 추천 알고리즘 개요 (1장)**

- ALS (Alternating Least Squares)
    
    - 잠재 요인 모델(Latent Factor Model)을 활용한 추천 모델
        
    - Matrix Factorization의 일종으로 사용자-상품 행렬 분해
        
- BPR (Bayesian Personalized Ranking)
    
    - 순위 학습(Ranking Learning) 기반의 개인화 추천 알고리즘
        
- K-means Clustering
    
    - 고객 군집화를 통해 유사한 고객군을 찾는 비지도 학습 알고리즘
        

---

### **5. 각 추천 시스템 동작 원리 상세 분석 (3장, 각 1장씩)**

_각 슬라이드에서 코드 분석 결과를 직관적으로 설명 (구체적 코드 자체가 아닌 흐름 중심)_

① **상품군 보유고객 분석 기반 추천 (Up-selling)**

- K-means로 고객 군집화
    
    - 저장된 K-means 모델 활용
        
- 각 군집별로 ALS를 통해 상품 추천
    
- ALS 하이퍼파라미터(alpha 등)는 고정된 값 사용 (근거 부족)
    

② **구매패턴 분석 기반 교차추천 (Cross-selling)**

- ALS 모델 기반으로 교차 추천 진행
    
- 사용자 구매 데이터를 기반으로 아이템 간 연관성 학습
    
- 하이퍼파라미터(alpha 등) 고정값 사용 (실험 근거 없음)
    

③ **카드사용패턴 분석 기반 체크카드 추천**

- ALS 기반으로 업종별 카드 사용 패턴을 학습하여 추천
    
- 업종별 소비 특성을 반영한 개인화 추천
    
- 역시 고정된 하이퍼파라미터 사용
    

---

### **6. 현재 모델 사용 방식의 공통적 한계 (1장)**

- 사전 저장된 모델 활용 → 실시간 고객 특성 반영 어려움
    
- 하이퍼파라미터 선정 근거 미흡 (최적성 미확인)
    
- 모델 성능 평가 지표나 모니터링 체계 부재
    
- 현재 코드 구조상 유지보수나 확장 어려움
    

---

### **7. 모델 한계점에 대한 개선 방안 제안 (1장)**

- 정기적이고 자동화된 하이퍼파라미터 튜닝 도입 필요
    
- ALS 외에 BPR과 같은 최신 개인화 추천 알고리즘 실험 필요
    
- 모델 성능 평가와 실험 결과 기록을 위한 체계적 모니터링 시스템 구축 필요
    
- 유연한 코드 구조 리팩토링 및 MLOps 도입 검토 필요
    

---

### **8. 결론 및 다음 단계 (1장)**

- 현재 추천 모델의 원리와 한계 이해를 통해 향후 발전 방향 설정 가능
    
- 데이터 및 아키텍쳐 파트와의 협업을 통해 모델 성능 향상 추진
    
- 이후 단계로 MLOps 기반 자동화 추천 서비스 전환 제안 예정
    

---

### **부록 (선택사항)**

- ALS, BPR, K-means 모델의 추가 이론적 배경
    
- 실제 코드 구조 예시(코드 흐름도 등)
